# x402 paid services

HISS exposes a set of **x402** endpoints — agent-discoverable, pay-per-call services
that compile, validate, score, simulate, and audit. They are **compute services**: they
return artifacts (coils, audits, receipts, risk scores), never execution. An agent
discovers an endpoint's price and schema, pays per call, and receives a verifiable
result.

## What x402 is here

[x402](https://www.x402.org/) is an open, HTTP-native pay-per-call standard. A HISS
x402 endpoint advertises its price and input schema so an agent can call and pay for it
programmatically. HISS uses x402 for **metered compute** — the "work the software does"
— not for anything custodial and not for trade execution.

## Representative endpoints

Compile / validate / score:

- `hiss-coil-compiler`, `compile-coil-for-robinhood`, `compile-vault-rebalance-policy`
- `validate-usdg-vault-fuses`, `validate-autonomy-fuses`
- `vault-risk-score`, `hiss-risk-audit`

Simulate / audit / receipts:

- `simulate-rebalance-plan`, `simulate-vault-rebalance`
- `post-run-audit`, `post-vault-rebalance-audit`
- `generate-vault-receipt`, `hiss-receipt-verify`
- `hiss-drift-check`, `hiss-robinhood-capsule`

Reports / packs:

- `vault-comparison-report` (facts-only, no recommendation)
- `vault-creator-due-diligence-pack`
- `generate-bankrbot-trading-prompt`

Each endpoint ships an x402 descriptor (e.g. `bankr.x402.json`) with its price and
schema.

## Discovery and calling

An agent reads the endpoint's x402 descriptor (price + input schema), constructs the
payment, and calls it. The response is a structured artifact the agent can validate.
Prices are per-call and advertised; there is no subscription requirement to call a
single endpoint.

## Safety model

- **Compute, not execution.** Endpoints return plans, scores, audits, and receipts.
  None places a trade, moves funds, or takes custody.
- **No credentials.** Endpoints reject credential-shaped fields; they never accept a
  private key or brokerage secret.
- **Execution-claim guard.** Outputs pass the honesty guard — no result can claim an
  order was placed.
- **Facts-only scoring.** Risk scores and comparisons use facts, never PnL/APY, and
  carry no recommendation.

## Honesty rules

- Never claim "Robinhood supports x402."
- A simulation or audit is **not** a forecast and **not** a performance claim.
- Region and provider constraints on downstream rails still apply — an endpoint that
  compiles a Bankr/Robinhood path does not make that path universally available.
