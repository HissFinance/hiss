# Fees — Overview

This is the complete, authoritative guide to every fee in HISS Finance. It exists so
you never encounter a surprise charge: **every fee is disclosed, and nothing is
hidden**. There are two independent fee systems, and they must never be conflated:

1. **[Vault fees](./vault-fees.md)** — what a creator and depositors pay around a
   USDG vault (creation, performance, protocol share, routing; deposit/withdrawal
   are zero). Denominated in **USDG (6 decimals)**.
2. **[$HISS token trading fees](./hiss-token-fees.md)** — fees generated by trading
   the $HISS token itself on its launch liquidity pool, accruing in **$HISS** and
   **WETH**. A verified portion of these funds the **[reward flywheel](./reward-flywheel.md)**.

## How to read every fee in these docs

Each fee is documented with the same fields so you can verify it independently:

| Field                           | Meaning                                                                          |
| ------------------------------- | -------------------------------------------------------------------------------- |
| **Source**                      | Where the value comes from (contract, launch schedule, policy).                  |
| **Exact current value**         | The number in effect at launch.                                                  |
| **Version**                     | The versioned schedule/policy the value belongs to.                              |
| **Where verifiable**            | How to confirm it (contract read, source constant, chain event).                 |
| **Contract-enforced vs policy** | Whether a smart contract enforces it, or it is a published policy/preview value. |
| **Example**                     | A worked calculation.                                                            |
| **Caveat**                      | What could change it, and what it is _not_.                                      |

> **Candidate vs contract-fixed.** USDG amounts such as the 50 USDG creation fee are
> **launch-schedule candidates**: the deployed `VaultFactory` is the source of truth
> at execution time. Percentage caps enforced in the contracts (for example the
> performance-fee bound) are contract-fixed. Each page labels which is which.

| Fee                     | Value (launch)                          | Enforced by                               | Denomination |

## Copy rules these docs follow

- No "guaranteed yield", "APY", "projected APY", "passive income", "dividends",
  "risk-free", or external-audit claims.
- Fee illustrations are **static fee math, not forecasts and not performance
  claims**.
- Vaults share **profits and losses**. There is no floor.

## Where the money is _not_ touched

- **Treasury assets are separate** from reward flows. Treasury holdings are not
  depositor funds and are not part of any split beyond the explicit treasury legs.
- **The 15% creator premint** of $HISS (2-year vest, 30-day cliff) is **always
  excluded** from reward flows.
- **Depositor principal** is never charged a performance fee — the protocol share is
  carved from the _creator's_ fee, never from principal.
